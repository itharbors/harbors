# SQLite Kit

SQLite Kit 提供一个面向真实数据库工作的中文工作台，包括受控文件选择、数据浏览、结构查看、记录编辑、导出和 SQL 执行。

## 连接与安全

- 打开已有数据库时默认使用只读连接；需要编辑时，必须在工作台中显式确认“启用写入”。
- 新建数据库使用可写连接。未填写扩展名时自动补充 `.sqlite`，且不会覆盖已有文件。
- 文件浏览器默认只显示文件夹和 `.sqlite`、`.sqlite3`、`.db` 文件。
- 普通表、视图、虚拟表和 SQLite 影子表会分组显示。视图、虚拟表、影子表及没有稳定行标识的表禁止使用记录编辑功能。

## 数据与结构

- 数据页固定使用 25 或 50 条分页，并以主键或 `rowid` 保证稳定排序。
- 支持快速搜索、列筛选、列排序、完整单元格查看、复制整行，以及按当前查询条件导出 CSV/JSON。
- 单次导出最多 10,000 条，超过上限时返回截断提示。
- 新增、编辑和删除均经过写入保护；最近一次成功操作可在 10 秒内撤销。
- 结构页展示字段、索引、外键、触发器和格式化 DDL，并支持复制和换行切换。

## 全库关系图

- “关系图”页签展示普通表、虚拟表、字段、主键和 SQLite 已声明的外键；视图与 SQLite 影子表不会混入关系图。
- 没有外键的表仍会作为独立节点出现。关系来自数据库声明，不根据字段名称推断。
- 使用搜索高亮表，拖拽平移画布，滚轮或工具栏缩放，并可一键适应窗口。
- 点击或使用键盘激活表节点，会跳转到该表的“结构”页查看完整定义。

## SQL 工作区

- 支持格式化、关键字/对象补全、查询计划、20 条会话历史及结果复制/导出。
- 写 SQL 会先分析风险并要求确认；只读连接不会获得写入确认令牌。
- 查询结果最多渲染 50 行。执行中的 SQL 可取消，取消后仍可继续运行下一条语句。
