# 架构决策记录

架构决策记录（ADR）保存重要技术取舍的上下文、选择、替代方案和后果。架构文档说明
“当前如何工作”，ADR 说明“为什么选择这样工作”。

## 编号与状态

- 文件名：`NNNN-short-title.md`，编号连续递增。
- “提议”：正在讨论，不能作为已实现行为的唯一依据。
- “已接受”：当前团队选择，应在架构文档和实现中保持一致。
- “已取代”：由新 ADR 替换；旧文档保留并双向链接。
- “已拒绝”：保留讨论结果，避免重复提出相同方案。

## ADR 模板

```markdown
# ADR NNNN：标题

- 状态：提议
- 日期：YYYY-MM-DD
- 决策者：项目维护者

## 上下文

描述问题、约束和必须作出决策的原因。

## 决策

写出一个可验证的选择。

## 替代方案

列出认真考虑过的方案及未采用原因。

## 影响

分别列出正面、负面和后续约束。

## 关联

链接架构文档、代码入口和取代关系。
```

修改 ADR 前阅读[文档维护指南](../guides/maintaining-docs.md)。
