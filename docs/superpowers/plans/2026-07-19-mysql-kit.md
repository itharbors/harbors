# MySQL Kit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a standalone MySQL Kit that connects to one database, previews schema and paginated rows, performs safe record CRUD, and executes explicit single-statement SQL.

**Architecture:** Add one kit-scoped `@itharbors/mysql-workbench` plugin. Its server-side main owns an async `MysqlService` behind a small injectable driver boundary implemented with `mysql2/promise`; its browser panel owns workbench state and calls plugin requests. Keep connection secrets server-only, preserve MySQL values through a JSON-safe protocol, and allow row updates/deletes only when a stable primary-key identity exists.

**Tech Stack:** TypeScript, `mysql2` 3.23.x, CE Editor kit/plugin runtime, DOM APIs, CSS, Vitest, jsdom, disposable MySQL 8.4 integration database.

## Global Constraints

- Create a separate `kits/mysql`; do not change the SQLite Kit into a multi-database Kit.
- Support one MySQL connection and one selected database per editor session.
- Connection passwords remain in the Server-side driver only and are never returned, logged, or persisted.
- Default host is `127.0.0.1`, port is 3306, pool limit is 4, and connection timeout is 10 seconds.
- TLS uses system trust and certificate verification; there is no insecure certificate bypass.
- Default page size is 100; supported sizes are 25, 50, 100, and 250; 250 is the hard maximum.
- Ordinary tables may insert rows. Update and delete require all primary-key columns; views are fully read-only.
- BINARY/BLOB values are preview-only summaries in the first version.
- Generated writes quote identifiers, bind every value, run in transactions, and require delete confirmation in the Panel.
- The SQL console executes one explicit statement with `multipleStatements: false` and previews at most 500 rows.
- Do not add generic MySQL HTTP routes or modify shared editor runtime contracts.

- [ ] **Step 3: Implement exact protocol unions and validators**

Define:

```ts
export type SerializedValue =
  | null | string | number | boolean
  | { type: 'integer'; mysqlType: 'BIGINT' | 'BIGINT UNSIGNED'; value: string }
  | { type: 'decimal'; value: string }
  | { type: 'date' | 'time' | 'datetime' | 'timestamp'; value: string }
  | { type: 'json'; value: string }
  | { type: 'blob'; size: number; previewHex: string };

export type EditableValue =
  | { type: 'null' }
  | { type: 'integer' | 'decimal' | 'real' | 'text' | 'date' | 'time' | 'datetime' | 'timestamp' | 'json'; value: string };

export type ConnectionInput = {
  host: string; port: number; user: string; password: string; database: string; tls: boolean;
};
```

`deserializeEditableValue` returns `null`, a finite number for `real`, and the validated string for every other type. Integer uses `/^[+-]?\d+$/`; decimal accepts a finite decimal/exponent lexical form; JSON must parse successfully but returns its original string. `serializeMysqlValue(value, mysqlType)` uses the field type to distinguish text from numeric/temporal strings, rejects non-finite numbers, and returns the first 16 BLOB bytes as lowercase hex. `parseConnectionInput` trims host/user/database but never password. `parsePageInput` permits exactly 25, 50, 100, or 250.

export type DriverResult =
  | { kind: 'rows'; rows: unknown[][]; fields: Array<{ name: string; mysqlType: string }> }
  | { kind: 'mutation'; affectedRows: number; insertId: string; warningStatus: number };

export interface MysqlExecutor { query(sql: string, values?: readonly unknown[]): Promise<DriverResult> }
export interface MysqlConnection extends MysqlExecutor {
  beginTransaction(): Promise<void>; commit(): Promise<void>; rollback(): Promise<void>; release(): void;
}
export interface MysqlPool extends MysqlExecutor { getConnection(): Promise<MysqlConnection>; end(): Promise<void> }
export interface MysqlDriver { createPool(input: ConnectionInput): MysqlPool }
