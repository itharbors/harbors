# SQLite 全库表关系图设计

## 背景与目标

SQLite Kit 已能在单表结构页展示外键，但用户无法从数据库整体视角理解表之间的关系。本功能新增数据库级“关系图”页签，以类似 Mermaid ER 图的方式展示全部用户表、字段和已声明外键，并提供适合大型画布的浏览能力。

关系图只读取数据库元数据，不修改数据库。首版聚焦结构理解，不提供可视化建表、拖拽修改外键、视图依赖分析、关系推断或图片导出。

## 产品决策

### 范围

- 展示普通表和用户创建的虚拟表，包括没有任何外键的孤立表。
- 不展示 view 和 SQLite shadow table。视图依赖不等同于外键关系，shadow table 属于实现细节，加入图中会制造噪声。
- 关系仅来自 `PRAGMA foreign_key_list`。不根据 `user_id` 等字段命名猜测关系。
- 表节点展示所有可见字段的名称和声明类型，并标记主键 `PK` 与外键 `FK`。隐藏列不展示。
- 复合外键按 SQLite 的同一 `id` 聚合成一条关系，并保留按 `sequence` 排序的完整字段映射。
- 自引用、循环引用、同一对表之间的多条关系都必须保留。

### 入口与导航

工作区页签调整为“数据 / 结构 / 关系图 / SQL”。“关系图”是数据库级页签：连接数据库后即可使用，不要求当前已选中对象；空数据库也可进入。

关系图激活时，工作区标题展示当前数据库而不是左侧已选对象。点击表节点会选中对应对象并切换到“结构”页，复用现有结构详情，而不是在图内增加第二套详情面板。

### 画布交互

关系图由顶部工具栏和可交互画布组成：

- 工具栏提供表名搜索、缩小、放大和“适应窗口”。
- 鼠标滚轮缩放，拖拽空白区域平移；缩放围绕指针位置进行。
- 默认首次加载执行“适应窗口”，后续数据库刷新或关系数据变化时重新适配。
- 搜索不隐藏其他表，而是高亮匹配节点、降低非匹配节点和关系的视觉权重，避免图的空间结构跳动。
- 缩放范围固定为 30%–200%，按钮和滚轮共用同一约束。
- 表节点可通过键盘聚焦，`Enter` 或 `Space` 跳转结构页。图外提供屏幕阅读器可读的关系列表摘要。

画布为空时展示明确状态：空数据库显示“此数据库还没有可展示的表”；存在表但没有声明外键时仍展示全部孤立节点，并提示“未检测到已声明的外键关系”。请求失败沿用工作台现有错误区和重试路径，不保留过期图。

## 技术方案

### 为什么不直接嵌入 Mermaid

SQLite Panel 当前通过 `tsc` 直接输出浏览器 ESM，没有 bundler，也没有运行时裸 npm import 解析。引入 Mermaid 会显著增加静态产物和构建复杂度，生成 DOM 的交互与主题也难以和现有工作台保持一致。

采用原生 HTML 表节点、SVG 关系连线和仓库内确定性布局算法：HTML 保证字段可读性与键盘语义，SVG 适合绘制连线，两者放在同一个带 CSS transform 的画布层中同步平移和缩放。该方案不新增运行时依赖。

### 服务协议

`SqliteService` 新增 `getRelationshipGraph()`，通过插件 request 一次返回完整模型，避免 Panel 对每个表逐次调用 `getObjectSchema`：

```ts
type RelationshipGraph = {
  tables: RelationshipTable[];
  relationships: Relationship[];
};

type RelationshipTable = {
  name: string;
  kind: 'table' | 'virtual';
  columns: Array<{
    name: string;
    type: string;
    primaryKeyOrder: number;
    foreignKey: boolean;
  }>;
};

type Relationship = {
  id: string;
  fromTable: string;
  toTable: string;
  columns: Array<{ from: string; to: string | null }>;
  onUpdate: string;
  onDelete: string;
};
```

服务先读取 `getSchema()`，筛选 `kind` 为 `table` 或 `virtual` 的对象，再读取字段和外键。关系目标不在图中时不生成悬空边，但来源字段仍保留 `FK` 标记。关系 `id` 由来源表名和 SQLite 外键 `id` 构造，仅用于当前快照中的稳定渲染键。

协议使用真实字段名，不拼装 Mermaid 文本，因此特殊字符、空格和引号不需要额外语法转义。读取过程在单个同步 request 内完成，并沿用现有数据库连接和错误归一化边界。

### 前端模块边界

新增 `panel.workbench/src/relationship-view.ts`，集中处理：

- `RelationshipGraph` 前端类型；
- 确定性布局计算；
- 关系路径和节点视觉模型；
- 缩放、平移和适应窗口的纯状态转换；
- 关系图 DOM 创建与事件回调。

现有 `index.ts` 只负责生命周期和装配：维护关系图数据、加载状态和请求序列，调用渲染器，并在节点激活时更新 `selectedName`、切换到结构页并加载结构数据。关系图数据在连接切换、关闭或 schema 刷新时清空；异步响应必须同时校验 connection generation、request sequence 和当前页签，防止旧数据库响应覆盖新状态。

`copy.ts` 增加关系图页签、工具栏、空状态和状态栏文案。`index.css` 增加关系画布样式，并复用现有深色网格、teal 强调色、焦点轮廓和字号下限。

### 确定性布局

布局算法不依赖随机力导向，保证测试稳定且刷新时节点不会无故跳动：

1. 按表名进行稳定排序。
2. 将外键关系转换为父表到引用表的有向图。
3. 使用强连通分量折叠循环关系；同一分量中的表按名称排序。
4. 对折叠后的无环图按最长路径分层，父表位于左侧、引用表位于右侧。
5. 每层按相邻层重心和表名稳定排序，减少交叉。
6. 没有任何关系的表放在主图下方的独立网格中。
7. 节点宽度固定，节点高度由可见字段数决定；层间距与行间距固定。
8. 关系使用正交 SVG path。同表自引用使用节点右侧回环；平行边按稳定序号错开。

布局输出画布尺寸、节点位置和边路径，不读取 DOM 尺寸。字体、行高、节点内边距与计算常量共用导出配置，避免渲染尺寸与路径锚点漂移。

## 状态、性能与无障碍

- 关系图只在首次进入页签或 schema 刷新后请求，缩放和平移不触发服务端读取。
- 单次快照完整渲染所有表和可见字段，以满足“所有表关系图”的目标，不以分页或折叠替换完整结构。
- 拖拽期间只更新画布 transform，不重新计算布局或重建节点。
- 装卸载时清理 pointer 和 wheel 监听；遵循 `prefers-reduced-motion`，不添加持续动画。
- 工具栏按钮提供可见文本或 `aria-label`；节点使用可聚焦的按钮语义；SVG 连线为视觉装饰并 `aria-hidden`。
- 屏幕阅读器摘要逐条描述“来源表.字段 → 目标表.字段”，复合外键在同一条摘要内列出全部字段映射。
- 720px 以下沿用对象导航抽屉；关系图工具栏允许横向滚动，画布交互保持不变。

## 测试策略

### 服务测试

使用真实临时 SQLite 数据库覆盖：

- 普通表、虚拟表和孤立表全部进入节点；view 与 shadow table 不进入节点；
- 字段类型、复合主键、外键标记和隐藏列处理；
- 普通外键、复合外键、自引用、循环引用和平行外键；
- 特殊字符表名和字段名原样返回；
- 外键目标不在可展示节点中时不生成悬空关系；
- 未连接时返回稳定错误。

### 纯函数测试

`relationship-view` 测试覆盖：

- 相同输入产生完全相同布局；
- 父子分层、循环分量、孤立节点网格、自引用和平行边路径；
- 所有节点矩形互不重叠、所有节点位于画布范围内；
- 适应窗口、缩放上下限、围绕指针缩放和平移状态；
- 搜索匹配使用不区分大小写的表名比较。

### Panel 与运行时测试

- 连接后关系图页签可用，即使没有选中对象；数据与结构页仍要求选中对象。
- 进入页签只发起一次 `getRelationshipGraph`，刷新后重新加载，过期响应被忽略。
- 节点、PK/FK 标记、关系摘要、无关系提示和空数据库状态正确渲染。
- 搜索、缩放、适应窗口、指针平移和键盘激活节点可用。
- 激活节点会选中对象、切换结构页并请求对应结构。
- 窄屏下工具栏和画布保持可操作。
- 插件 manifest 暴露 request，main bridge 调用服务方法。

最终运行 SQLite Kit 测试、插件构建检查与仓库完整 `npm run check`。真实页面验收使用带普通、复合、自引用和孤立表的临时数据库，确认关系图首次适配、缩放平移、搜索高亮和节点跳转。

## 完成标准

1. 连接任意 SQLite 数据库后，可以在独立“关系图”界面看到全部用户表。
2. 所有已声明且目标可展示的外键都以可辨认连线出现，字段映射可被读出。
3. 无外键表不会消失；空库、无关系和错误状态有明确反馈。
4. 画布可搜索、缩放、平移和适应窗口，节点可跳转结构页并支持键盘。
5. 服务、布局、Panel、manifest 和真实页面路径均有对应验证证据，完整仓库检查通过。
