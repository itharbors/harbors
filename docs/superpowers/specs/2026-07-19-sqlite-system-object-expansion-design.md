# SQLite 系统对象展开状态设计

## 问题

系统对象分组使用 `details + summary` 并默认折叠。用户展开分组并选择其中的对象后，对象加载会触发完整界面重绘；当前渲染逻辑每次都把新建的 `details` 强制设为关闭，因此分组会立即收起，遮断用户对当前选择所在位置的认知。

## 已确认方案

- 在工作台状态中记录可折叠对象分组的展开状态。
- 系统对象在首次打开数据库时仍默认折叠，保持现有信息层级。
- 用户通过鼠标或键盘展开、折叠系统对象时，同步更新该状态。
- 选择系统对象、加载数据、切换页签、搜索对象及其他完整重绘不得改变用户选择的展开状态。
- 关闭数据库或打开另一数据库时重置分组状态，新连接仍从默认折叠开始。
- 普通表、视图和虚拟表继续保持不可折叠，不改变现有导航结构。

## 实现边界

渲染系统对象分组时，从工作台状态恢复 `details.open`，并通过 `toggle` 事件保存后续变化。对象选择仍沿用现有异步加载和完整重绘流程，不为此引入局部 DOM 更新或修改服务端协议。

## 验收与测试

1. 新连接中的系统对象默认折叠。
2. 展开系统对象并选择其中一个条目后，等待对象数据加载及重绘完成，系统对象仍保持展开。
3. 用户主动折叠后，后续重绘保持折叠。
4. 打开另一数据库后恢复默认折叠。
5. SQLite Panel 测试、插件检查和全仓检查通过。
