# MySQL 连接栏布局修复设计

## 问题

单 Kit 浏览器页面中，连接面板高度固定为 78px，但连接表单实际高度达到 92px；同时表单被限制在 617px 宽的中间列中，而其内容至少需要 721px。结果是输入框被垂直裁切，“断开连接”逐字换行，并与右侧状态说明重叠。

## 方案

连接栏改为 112px 高的两层布局。左侧品牌区跨两行；右侧第一行独占完整宽度展示连接字段和三个操作按钮，第二行展示连接状态、端点和会话提示。操作按钮禁止换行，并为“连接”和“断开连接”设置足够的最小宽度。

Kit 的初始分栏高度、面板最小高度和面板 CSS 使用同一个 112px 约束。窄于桌面工作区时仍保留现有横向滚动策略，不改变主工作区、对象栏或连接逻辑。

## 验证

- 自动化测试锁定 112px 分栏与面板最小高度，以及两层 CSS 网格和按钮不换行约束。
- 构建 MySQL Explorer 后，在 1092px 实际面板宽度下检查无垂直裁切、无控件重叠、无按钮文字换行。
- 运行仓库完整检查。
