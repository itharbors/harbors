# SQLite 单元格详情交互调整设计

## 背景

SQLite 工作台当前在单击任意数据单元格时打开右侧详情侧栏。普通短值无需额外详情，侧栏还会遮挡其他列，容易被误认为记录编辑器。

## 已确认方案

- 单击数据单元格只选中所在记录，不再打开详情侧栏。
- 仅当值为 BLOB，或格式化后的文本超过 80 个字符或包含换行时，显示“查看字段详情”按钮。
- 点击该按钮或双击同一可展开单元格时，打开字段详情侧栏。
- 字段详情明确标记为只读，保留字段名、完整文本（BLOB 为现有十六进制预览）、复制和关闭操作。
- 普通短值不显示展开按钮，双击时也不打开字段详情。
- 记录编辑入口不变：使用工具栏“编辑”、整行 Enter 或双击不可展开的普通单元格。

## 边界与可访问性

- 展开按钮必须有包含字段名的 `aria-label`，不能仅依赖图标表达含义。
- 点击展开按钮不得触发行选择以外的编辑动作。
- 详情侧栏不得在切换对象或关闭数据库后残留。
- 不改变数据库读写规则、序列化协议或服务端接口。

## 验收

1. 单击短值仅选中记录，页面不存在字段详情侧栏。
2. 长文本、含换行文本和 BLOB 显示唯一的显式展开入口。
3. 展开入口和双击长内容均打开带“只读”标识的字段详情。
4. 复制和关闭仍可用，短值不会意外打开详情。
5. SQLite Panel 测试、插件构建检查和全仓检查通过。
