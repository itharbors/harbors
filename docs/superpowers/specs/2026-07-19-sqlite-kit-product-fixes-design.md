# SQLite Kit 产品体验与安全修复设计

## 背景

SQLite Kit 已具备打开数据库、查看对象、分页浏览记录、记录 CRUD 和执行 SQL 的基础能力，但真实数据库走查暴露出三类问题：关键操作不易发现或无法可靠触发，内部对象和写 SQL 缺少安全边界，以及数据浏览、结构阅读、长查询和无障碍体验不完整。

本设计一次性覆盖走查中确认的全部问题。实现保持在 `kits/sqlite` 的插件边界内，不增加通用数据库 HTTP API，也不让 Panel 读取 Electron 或父页面私有对象。

## 目标与验收口径

完成后，用户应能：

1. 不输入绝对路径，通过 Kit 内文件浏览器打开或创建本机 SQLite 文件；浏览器和 Electron 使用同一套流程。
2. 默认以只读模式打开数据库，显式解锁后才能新增、编辑、删除或执行写 SQL。
3. 不接触 FTS shadow table 等内部表；系统对象默认折叠且始终只读。
4. 点击整行、行选择控件或使用键盘选中记录，随后可靠地编辑或删除。
5. 对数据进行稳定排序、快速搜索和列筛选，查看长单元格、复制内容并导出当前结果。
6. 阅读格式化、带语法颜色和行号的 DDL，并查看字段、索引、外键和触发器。
7. 使用带真实行号、格式化、对象补全、历史、查询计划和取消能力的 SQL 工作区。
8. 在每个破坏性操作前看到具体影响，并在 CRUD 后短时撤销。
9. 在常用桌面和窄屏尺寸下获得可读、可聚焦、可用键盘操作的界面。

## 产品决策

### 文件选择

采用插件内受控文件浏览器，而不是只接 Electron 原生对话框。Panel 调用插件 main 的文件系统查询接口，由 main 返回目录与 SQLite 文件条目。这样当前浏览器运行方式和 Electron 均可用，且不扩大 Panel 权限。

文件浏览器分为“打开”和“新建”两种模式：

- 打开模式只允许选择普通文件，优先展示 `.sqlite`、`.sqlite3` 和 `.db`，其他文件仍可在“显示全部文件”后选择并由 SQLite 校验。
- 新建模式选择目录并输入文件名；自动建议 `.sqlite` 后缀，拒绝覆盖已有文件。
- 初始目录依次取当前连接目录、当前 session 最近目录、用户主目录。
- 最近数据库只保存在当前 editor session 的服务实例内，不写入跨 session 的隐式历史文件。
- 手工路径输入保留在“高级”折叠区，便于粘贴路径和自动化测试。

### 连接与写入安全

`openDatabase` 增加连接模式：`readonly` 或 `readwrite`。打开现有数据库默认 `readonly`；新建数据库默认 `readwrite`。顶部只展示当前连接的完整规范化路径，待选择路径存在于文件选择器内，避免输入框与实际连接不一致。

只读连接下：

- 新增、编辑、删除按钮禁用并显示原因。
- SQL 服务在 main 边界拒绝非只读 statement，不能只依赖 Panel 判断。
- “启用写入”要求显式确认并重新以读写模式打开同一文件；失败时保留原只读连接。

读写连接下，普通表可写；视图、virtual table、shadow table 和无法生成稳定行标识的表保持只读。系统对象不因高级模式而变得可写。

### 数据浏览与记录操作

数据表采用单行选择模型：

- 整行点击选中，首列使用明确的单选选择控件；再次点击已选行保持选中。
- `ArrowUp`、`ArrowDown` 移动选择，`Enter` 打开编辑，`Space` 选择；双击不可展开的普通单元格打开编辑。
- 行使用 `aria-selected`，表格工具栏实时显示已选记录的主键或 rowid 摘要。
- 选中态与 hover 明显区分；选择控件和对象徽标满足小字号文本对比度要求。

分页默认 50 行，可选 25 或 50，单次 DOM 不渲染超过 50 行。服务端为每次读取生成稳定排序：优先完整主键，其次 rowid；用户可点击列头切换升序或降序，稳定标识作为末级排序键。

快速搜索在可显示的文本/数字列上执行参数化 contains 查询；列筛选支持“包含、等于、为空、非空”。当前排序和筛选在对象切换时重置，在页码变化时保留。

数据工具栏固定为两行：第一行承载记录操作、搜索、复制和导出，第二行承载列筛选。控件禁止文字折行；窄宽度下每行独立横向滚动，避免按钮被挤压成竖排。

单击单元格只选中记录，不自动打开详情。超过 80 个字符、包含换行或为 BLOB 的值显示“查看字段详情”按钮，并支持双击打开只读详情抽屉；抽屉可查看、换行并复制完整文本或 BLOB 预览。工具栏支持复制选中行，以及将当前筛选结果下载为 CSV 或 JSON；导出由 main 分批读取并设定固定最大行数，超出时给出明确提示。

### 破坏性操作与撤销

删除不再使用 `window.confirm`。自定义模态框展示数据库、表、主键/rowid 和记录摘要，并提示外键级联风险。SQL 写语句执行前展示语句类型和目标对象；DDL、无 WHERE 的 UPDATE/DELETE 额外使用高风险提示。

自动 CRUD 在服务内保存一次可逆快照：

- insert 的逆操作按新记录稳定标识删除；
- update 保存修改前的完整可编辑字段；
- delete 保存删除前的完整记录值，包括 BLOB。

Panel 在成功后显示 10 秒“撤销”入口。若并发修改导致无法安全撤销，服务拒绝并说明原因。SQL 控制台的任意写语句不承诺自动撤销，但仍受写入解锁与确认保护。

### 对象导航与结构页

对象列表增加搜索，并使用 `PRAGMA table_list` 区分 table、view、virtual 和 shadow：

- 普通表和视图默认展示。
- virtual table 单独分组并标记能力限制。
- shadow table 归入默认折叠的“系统对象”，始终只读。
- 对象数量、分组和只读原因都由服务返回的结构化字段驱动。

所有对象分组共用相同的标题、计数、字号和间距；系统对象只增加统一样式的折叠箭头，不使用浏览器原生 summary 外观。

结构页展示字段、主键、默认值、索引、外键和触发器。DDL 先经过确定性的 SQLite 格式化器，再以 token 渲染关键字、标识符、字符串、数字和注释；同时提供行号、复制和自动换行开关。格式化失败时安全回退为原文，绝不修改 SQL 内容。

### SQL 工作区

SQL 输入仍以原生 textarea 为可编辑源，避免引入大型编辑器依赖；周围增加：

- 与内容同步的真实行号；
- SQLite 关键字和对象名补全；
- 格式化、执行、查询计划、取消按钮；
- 当前 session 最近 20 条历史；
- 结果按 50 行分页，并支持单元格详情、复制和 CSV/JSON 下载。

SQL 执行移入 Node worker。worker 独立打开当前数据库文件并继承连接模式、外键和忙碌超时设置。main 持有当前 worker，可通过 `cancelSql` 终止；结束、取消或错误后保证清理 worker。写 SQL 只允许在 `readwrite` 模式且 Panel 传入 main 返回的分析确认令牌后执行，避免绕过确认。

### 模态框、错误和状态

所有对话框使用 `showModal()`，具备初始焦点、焦点循环、Escape 行为和未保存更改提醒。字段校验错误显示在字段旁，并聚焦首个错误。错误对象使用稳定 code 和中文用户信息；底层英文 SQLite 信息放在可展开详情，不与通用建议拼接成模糊文案。

顶部状态只展示“只读/可写、数据库名称、完整路径”；SQLite 版本、外键和忙碌超时移入连接详情。状态栏避免重复当前对象名称。

## 架构与文件边界

现有 `panel.workbench/src/index.ts` 已同时承担状态、渲染、事件和格式化职责。此次改造拆分为：

- `main/src/protocol.ts`：连接模式、对象类型、排序筛选、文件条目、SQL 分析与导出协议。
- `main/src/sqlite-service.ts`：连接、对象分类、稳定查询、CRUD、撤销和导出。
- `main/src/sql-worker.ts`：可取消 SQL 执行与结果序列化。
- `main/src/file-browser.ts`：目录枚举、路径校验、创建目标校验。
- `panel.workbench/src/state.ts`：单一 WorkbenchState 与纯状态转换。
- `panel.workbench/src/controller.ts`：request、异步动作、确认令牌和取消协调。
- `panel.workbench/src/data-view.ts`：表格、筛选、排序、选择、详情和导出。
- `panel.workbench/src/schema-view.ts`：结构分组与 DDL token 渲染。
- `panel.workbench/src/sql-view.ts`：编辑器、历史、补全和结果。
- `panel.workbench/src/dialogs.ts`：文件、记录、确认和详情模态框。
- `panel.workbench/src/sql-format.ts`：无副作用的 SQLite 格式化与 token 化。
- `panel.workbench/src/index.ts`：Panel 生命周期和模块装配。

拆分不改变 Kit manifest、通用 Server 路由或 Panel runtime 合约。

## 性能与无障碍

- 数据页和 SQL 结果每次最多渲染 50 行；导出不复用 DOM。
- 对象搜索在内存中执行；数据库读取继续参数化，并缓存相同筛选条件的计数结果，刷新或写入后失效。
- 正文不小于 12px，辅助信息不小于 11px；正文采用系统中文 UI 字体，路径、SQL、标识符和数值使用等宽字体。
- 所有图形装饰使用 `aria-hidden`；icon-only 控件提供 `aria-label`。
- tabs 实现 `aria-controls`、roving tabindex 和左右方向键。
- 异步状态使用 polite live region，破坏性错误使用 alert；焦点状态在深色主题下清晰可见。
- 断点覆盖 1180px、880px 和 720px；窄屏将对象导航折叠为抽屉，避免挤压数据表。

## 测试策略

所有行为按 TDD 实现，测试分为四层：

1. 纯函数测试：SQL 格式化、CSV、筛选协议、对象分类、行选择和键盘状态转换。
2. 服务真实数据库测试：只读阻断、shadow table 分类、稳定排序、筛选、撤销、导出、文件浏览和 SQL worker 取消。
3. Panel jsdom 测试：文件选择、写入解锁、整行选择、模态焦点、结构高亮、SQL 历史和错误保留。
4. 运行时与真实页面验收：通过真实 editor session 打开临时数据库，完成只读打开、解锁、选择、CRUD、撤销、结构、筛选、导出和 SQL 取消流程。测试必须操作 Panel iframe，而不是只调用插件 API。

最终运行 SQLite Kit 测试、Client 测试、Server 测试、插件构建校验和仓库 `npm run check`。使用知识库数据库时只操作 SQLite 在线备份副本；测试结束校验原库哈希并清理副本。
