# SQLite 与 MySQL Kit 中文化设计

## 目标

将 SQLite Workbench 与 MySQL Workbench 的用户可见界面完整翻译为简体中文，同时保持数据库技术信息准确、可排障。

## 范围

中文化内容包括：

- 页面标题、品牌副标题、连接表单、按钮、页签和辅助说明；
- 对象列表、数据分页、结构信息、SQL 执行结果和记录编辑弹窗；
- 空状态、加载状态、成功状态、确认提示、输入校验和产品控制的错误信息；
- HTML 的页面语言声明和无障碍标签。

以下内容保持原样：SQLite、MySQL、SQL、TLS、NULL、BLOB 等技术术语，数据库对象名、字段名、字段值、SQL 文本、错误码以及数据库驱动返回的原始诊断。

## 结构

每个 Workbench 在自己的面板目录中维护独立的 `copy.ts` 文案模块。模块集中导出静态中文文案和少量动态文案函数；面板渲染与交互代码只负责选择文案和传入计数、名称等数据。

SQLite 与 MySQL 不共享文案模块，避免插件构建和产品术语互相耦合。暂不引入语言切换或完整 i18n 框架。

## 错误处理

产品生成的校验和操作提示使用中文。底层数据库或驱动返回的诊断信息不盲目翻译：保留错误码与原始详情，并在需要时添加中文操作建议，兼顾可读性和排障能力。

## 测试与验收

先在两个面板测试中加入中文界面和动态提示断言，确认其因现有英文文案而失败，再实现文案模块和替换。

验收条件：

- SQLite 与 MySQL 面板的主要流程不再出现产品控制的英文界面文案；
- 技术术语、用户数据和数据库原始诊断未被错误翻译；
- 两个 Kit 的完整测试通过，插件构建成功；
- 在开发服务中检查 SQLite 页面，并在切换 Kit 后检查 MySQL 页面布局和关键文案。
