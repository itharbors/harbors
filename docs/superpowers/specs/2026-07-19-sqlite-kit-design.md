# SQLite Kit 设计

## 目标

新增一个独立的 SQLite Kit，让用户可以打开或创建本地 SQLite 数据库，预览数据库结构和表数据，并完成常见的记录新增、修改、删除、查询操作。Kit 同时提供 SQL 控制台，用于执行超出表格操作能力的查询、DDL 和 DML。

首版聚焦单个本地数据库文件的桌面工作流，不提供远程数据库、数据库同步、迁移管理、可视化建表器或 BLOB 编辑器。

## 用户工作流

1. 用户使用路径输入框打开已有 `.db`、`.sqlite` 或 `.sqlite3` 文件；路径不存在时可确认创建新数据库。
2. 连接成功后，左侧导航展示数据库中的表和视图。
3. 选择表后，工作区默认打开数据页并分页加载记录。
4. 用户可以新增记录、编辑选中记录或删除选中记录；写入成功后刷新当前页和对象摘要。
5. 用户可以切换到结构页查看字段、主键、默认值和索引。
6. 用户可以在 SQL 页输入并主动执行 SQL，查看结果集、变更行数、执行耗时或错误。
7. 用户切换数据库或退出 Kit 时，当前数据库连接被关闭。

## 架构

创建 `kits/sqlite`，其中包含一个 `@itharbors/sqlite-workbench` 插件。Kit 使用标准 `package.json`、`layout.json`、`main.html` 和 `secondary.html`，不修改通用 Server 路由。

插件包含两个运行边界：

- `main` 在 Server 进程中运行，持有 `better-sqlite3` 连接，负责文件校验、schema 读取、SQL 生成、参数绑定、事务和错误归一化。
- `panel.workbench` 在浏览器中运行，负责路径输入、对象导航、分页表格、记录编辑表单、确认对话框和 SQL 控制台。Panel 只通过插件 request 调用 Server 能力，不直接访问文件系统或数据库。

单个插件足以覆盖首版工作流，也避免多个插件之间同步连接、选中对象和分页状态。数据库服务逻辑与 Panel 渲染逻辑分别放在独立源码文件中，便于单元测试和后续拆分。

## Kit 布局与界面

Kit 使用一个占满主窗口的 workbench panel。Panel 内部布局分为：

- 顶部连接栏：数据库路径、打开/创建、刷新、关闭按钮，以及当前连接状态。
- 左侧对象导航：表和视图分组，显示对象名称和近似行数；支持刷新。
- 右侧工作区：数据、结构、SQL 三个页签。
- 底部状态区：当前对象、分页范围、执行耗时、变更行数和最近错误。

数据页包含工具栏、横向可滚动的数据表格和分页器。新增与编辑使用模态表单：每个字段可选择 NULL；INTEGER/REAL 输入先进行数值解析；TEXT 保留字符串；BLOB 只展示摘要并在首版中禁止编辑。删除操作必须二次确认。

结构页展示 `PRAGMA table_xinfo`、`PRAGMA index_list` 和 `PRAGMA index_info` 的归一化结果。视图明确标注为只读，并展示其定义 SQL。

SQL 页包含编辑区、执行按钮和结果区域。结果区域复用数据表格展示查询结果；非查询语句显示变更行数。首版不实现自动补全或查询历史。

## 插件接口

插件 main 暴露以下 request：

- `getConnectionState()`：返回当前文件、连接状态和数据库版本。
- `openDatabase(input)`：打开已有文件，或按显式 `create` 标记创建文件。
- `closeDatabase()`：关闭当前连接并清空选择状态。
- `getSchema()`：返回表和视图摘要。
- `getObjectSchema(input)`：返回字段、主键、索引和定义 SQL。
- `getRows(input)`：按对象、页码和每页条数返回记录、列信息和总数。
- `insertRow(input)`：插入一条记录。
- `updateRow(input)`：根据稳定行标识更新一条记录。
- `deleteRow(input)`：根据稳定行标识删除一条记录。
- `executeSql(input)`：执行用户提交的 SQL 并返回结果集或变更摘要。

输入对象在 main 边界进行完整校验。对象名不作为 SQL 参数值拼接，而是通过严格的标识符引用函数双引号转义。记录值始终使用 SQLite 参数绑定。

## 数据读取与行标识

分页查询默认每页 100 行，可选 25、50、100 或 250，上限固定为 250。总数通过单独的 `COUNT(*)` 查询获得。首版不提供任意筛选器或排序构造器；复杂查询使用 SQL 控制台。

表格操作需要稳定行标识：

1. 有主键时，使用所有主键列组成复合标识。
2. 无显式主键且表支持 rowid 时，在内部查询中附加 `rowid`，但不把它伪装成普通字段。
3. `WITHOUT ROWID` 表必须有主键；若元数据异常或无法生成稳定标识，则数据仍可预览，但编辑和删除禁用。
4. 视图始终只读。

更新和删除语句的 WHERE 条件来自加载记录时保存的原始行标识，而不是用户修改后的字段值。操作必须检查变更行数恰好为 1；否则回滚并提示记录可能已变化。

## 类型与序列化

SQLite 值在插件协议中归一化为 JSON 可传输形式：

- `null`、字符串和有限数字保持原值。
- 超出 JavaScript 安全整数范围的整数以带类型标签的十进制字符串返回。
- BLOB 以 `{ type: "blob", size, previewHex }` 摘要返回，不传输完整内容。
- SQL 控制台参数首版不单独提供；用户输入的 SQL 由 SQLite 解析。

表单提交的数据包含显式类型和值，避免空字符串、NULL 和数字之间发生隐式混淆。

## 写入、安全与错误处理

自动生成的新增、修改和删除操作使用参数绑定，并在事务中执行。删除必须由 Panel 二次确认。SQL 控制台只有在用户点击执行或使用明确快捷键时运行，不在输入期间自动执行。

数据库文件路径会解析为绝对路径。打开已有文件时要求目标是普通文件，并通过 SQLite 连接验证格式；创建时要求父目录存在且目标路径尚不存在，避免误覆盖。一个 session 同时只持有一个连接。

数据库错误返回稳定的错误码与可读信息，例如未连接、路径无效、对象不存在、对象只读、约束失败、数据库锁定和 SQL 语法错误。Panel 保留当前输入并在状态区显示错误，不因单次失败清空用户工作。

切换数据库采用先验证新连接、再关闭旧连接的顺序；新连接失败时旧连接保持可用。插件卸载时关闭连接，关闭操作必须幂等。

## 测试与验收

Server 逻辑使用临时目录中的真实 SQLite 文件进行测试，至少覆盖：

- 打开已有数据库、创建新数据库、拒绝目录和无效数据库文件；
- 枚举表/视图并读取字段、主键和索引；
- 分页读取以及 NULL、数字、文本、大整数和 BLOB 的序列化；
- 有主键、复合主键、rowid 和 `WITHOUT ROWID` 表的行标识；
- 新增、修改、删除成功路径及唯一约束、外键约束和并发变更失败；
- 查询 SQL、写 SQL、DDL、语法错误和锁定错误；
- 切换连接失败时保留旧连接，以及关闭/卸载幂等。

Panel 测试至少覆盖：

- 未连接、加载中、空数据库、空表和错误状态；
- 对象选择、页签切换、分页和刷新；
- 新增/编辑表单的类型转换与 NULL 处理；
- 只读视图禁用写操作、删除确认和请求失败后保留输入；
- SQL 查询结果与非查询执行摘要。

最终验收还包括插件构建检查、项目类型检查和现有完整测试套件。使用 `npm run dev -- --kit ./kits/sqlite` 启动后，应能对临时数据库完成结构预览、分页查询和一轮新增、修改、删除操作。

## 非目标与后续扩展

首版不包含 schema 可视化编辑、数据库迁移、导入导出、数据过滤器、排序构造器、完整 BLOB 查看/编辑、加密 SQLite 扩展或多数据库标签页。这些能力可在已定义的 main request 和工作区页签基础上继续扩展，而无需改变 Kit 或通用 Server 架构。
