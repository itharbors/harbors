# SQLite 导航分组与工具栏布局设计

## 问题

对象导航中的普通分组使用 `section + h2`，系统对象使用浏览器原生 `details + summary`。后者继承默认三角形和排版，因此与“表”“虚拟表”的标题样式不一致。数据工具栏则被限制为 42px 单行，空间不足时按钮被压缩并发生中文逐字换行。

## 已确认方案

### 对象分组

- 所有对象分组共用 `.object-group-title` 标题组件，统一字号、字重、间距、颜色和“名称 · 数量”格式。
- 系统对象继续使用 `details` 并默认折叠，避免内部影子表抢占导航空间。
- 系统对象标题只额外显示一个自绘展开箭头；隐藏浏览器原生 marker，避免平台差异。
- 普通表、视图和虚拟表的展开状态与现状一致，不新增无意义的折叠操作。

### 数据工具栏

- 工具栏固定为两行，不允许根据空间任意增加行数。
- 第一行依次放置新增、编辑、删除、搜索、复制、CSV/JSON 导出和记录总数。
- 第二行依次放置筛选列、运算符、筛选值、应用筛选和清除筛选。
- 控件文字禁止换行和挤压；极窄宽度下每一行独立横向滚动，不把按钮排成竖排。
- 表格内容区域随工具栏高度调整，分页栏保持不变。

## 视觉约束

继续使用现有深色数据工作台的颜色、系统中文字体和等宽数据字体，不增加装饰性色彩。分组箭头使用当前青绿色强调色，工具栏两行之间用现有网格线分隔。

## 验收

1. “表”“视图”“虚拟表”“系统对象”标题拥有同一标题类和计数格式。
2. 系统对象默认折叠、可通过键盘和鼠标展开，且不显示浏览器默认 marker。
3. 数据工具栏恰好包含两个有语义的行容器，操作与筛选不混排。
4. 所有按钮及选择控件保持单行文字；1152px 视口下不发生竖排或第三行。
5. Panel、无障碍、SQLite 和全仓检查通过。
