# SQLite Explorer 文件弹窗回归修复设计

## 背景

SQLite Kit 拆分为六个插件后，打开和新建数据库的文件弹窗迁入左侧 Explorer Panel。Panel 运行在只有约 300px 宽的 iframe 中，因此 iframe 内的固定定位或原生模态框都无法覆盖整个编辑器，弹窗被限制在左栏。拆分同时遗漏了原有 `manualPath` 状态与手动路径输入控件。

## 目标

- 打开或新建数据库时，文件弹窗在整个编辑器工作区居中，宽度足以浏览路径和文件列表。
- 弹窗始终显示可编辑的数据库路径输入框；列表选择会回填路径，手动输入路径优先于列表选择。
- 取消弹窗、完成连接或卸载 Panel 后，Explorer 必须退出模态态并恢复原布局；连接失败时保留弹窗，允许用户修正路径后重试。
- 保持 Panel 的权限边界：Explorer 不直接读取或修改父页面私有 DOM。

## 方案

### 宿主模态态

Panel runtime 在 `context.panel` 上提供 `setModalOpen(open)`。它只向父页面发送结构化的 `ce-panel-modal-state` 消息，不传递 HTML 或脚本。

Editor App 仅接受来源为当前已挂载 Panel iframe 的消息，并在对应 `ce-panel` 上切换 `modal-open` 属性。Panel 组件使用该属性临时固定到编辑器视口、提升层级、隐藏 Panel 标题栏并让 iframe 占满覆盖层。关闭时移除属性。

该能力只改变承载 Panel 的几何范围，不接管插件状态、内容或业务事件。未受信来源和无效 payload 会被忽略。

### Explorer 文件弹窗

`FileDialog` 恢复 `manualPath`。文件弹窗在当前文件夹行与文件列表之间常驻一个“数据库路径”输入框：

- 打开模式的占位示例为 `/path/to/database.sqlite`。
- 新建模式的占位示例为 `/path/to/new-database.sqlite`。
- 选择文件时把文件绝对路径同步到输入框；继续浏览目录时清空旧选择，但保留用户手动输入。
- 确认时优先使用去除首尾空白后的手动路径；为空时，打开模式使用列表选择，新建模式使用当前目录和文件名。

成功载入目录并打开文件弹窗后进入宿主模态态；取消、成功连接、连接状态重置和 `unmount` 都退出模态态。连接失败时保持模态态和输入内容，错误仍在现有 Explorer 错误区域显示，不扩大文件系统权限。

## 视觉与交互

延续现有 SQLite 深色工作台样式，不引入新主题。弹窗目标宽度为约 680px，并在窄视口下保留 16px 安全边距；文件列表占主要垂直空间，路径输入使用等宽字体以便核对。键盘焦点样式沿用青绿色高对比描边，确认按钮在没有任何有效目标时禁用。

## 测试

- Client：可信 Panel 消息切换对应 `modal-open`，未受信消息无效；Panel 模态样式覆盖视口并隐藏标题栏。
- Server：注入的 Panel runtime 暴露 `panel.setModalOpen` 并发送结构化消息。
- SQLite Explorer：打开弹窗进入模态态；取消和卸载退出；列表选择回填路径；手动路径优先并传给 `openDatabase`。
- 真实页面：在 1253×1204 视口确认弹窗相对整个编辑器居中且宽度可用，路径输入可见并可编辑，关闭后左栏恢复。
