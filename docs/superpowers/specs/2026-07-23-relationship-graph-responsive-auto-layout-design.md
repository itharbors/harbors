# SQLite 与 MySQL 关系图自适应自动布局设计

## 背景与问题

SQLite 与 MySQL Kit 已共享 `@itharbors/relationship-graph`。当前自动布局会依据画布纵横比打包多个业务分组，但每个分组内部始终采用从左到右的层级：同层表固定从上到下堆叠。当多数表因名称或外键进入同一分组时，外层打包没有选择空间，宽画布仍得到狭长的纵向图，点击“自动排列”也无法利用页面宽度。

## 竞品结论

- DBeaver 的 Auto-arrange 目标是紧凑、可读的表排列。
- DataGrip 把布局与“布局后适应当前图区域”组合使用，并允许布局方向变化。
- ELK 暴露目标纵横比、方向和矩形打包优化目标；其中最大可用缩放优先的策略最符合关系图字段可读性。
- yFiles 的组件布局会尝试多行排列，使整体纵横比接近目标区域。

因此本次不只在布局后缩放一张固定形状的图，而是让布局本身根据当前画布宽高选择形状，再执行适应窗口。

## 目标

- 点击“自动排列”时读取当前关系画布尺寸，并让单个业务分组也能利用宽或高方向的空间。
- 宽画布倾向横向展开同层节点，窄高画布倾向纵向展开同层节点。
- 首次无缓存布局继续使用同一算法；已有手工布局不因窗口 resize 自动重排。
- 保留外键层次、确定性、节点不重叠、缓存、拖动、搜索、边路由和 SQLite/MySQL 共用实现。
- 以节点可读缩放为第一优化目标，纵横比接近度和占用面积作为并列判定，不靠无限放大少量节点伪造“填充”。

## 方案比较与选择

### 方案一：只调整最终 viewport

实现最小，但只能缩放既有纵向图，无法使用宽画布的空白；卡片还会更小。放弃。

### 方案二：固定方向下对同层节点换行

能缓解超长层，但需要同时决定每层列数和层间宽度，容易破坏层次边界，边路由也更拥挤。暂不采用。

### 方案三：为每个分组生成横向层级与纵向层级候选，再按当前画布评分

横向层级把 rank 放在不同列、同 rank 纵向排列；纵向层级把 rank 放在不同行、同 rank 横向排列。两种布局都保留强连通分量压缩后的 rank 顺序。候选评分优先比较适应画布后的可读缩放，再比较布局与画布纵横比误差、空白率和稳定方向偏好。它能直接解决单分组退化问题，改动边界集中且确定，因此采用。

## 详细设计

### 组内候选布局

`layoutGroup` 接收当前画布，基于同一份 `layers` 生成两个候选：

- `left-to-right`：沿 x 轴递增 rank，每个 rank 内沿 y 轴排列。
- `top-to-bottom`：沿 y 轴递增 rank，每个 rank 内沿 x 轴排列。

每个 rank 的跨度使用该 rank 的真实最大节点宽高计算，不能假设所有表等高。没有外键方向的孤立节点仍按自然名称组成局部网格；其列数从画布纵横比推导，避免固定平方根在极宽/极高页面上退化。

候选节点从局部原点开始，统一计算包围盒。相同输入和画布必须得到完全相同的位置；评分相同时使用稳定方向顺序，宽画布偏向 `top-to-bottom`，窄高画布偏向 `left-to-right`。

### 评分与整体打包

候选评分采用三层顺序：

1. 计算候选放入画布后的 `fitScale = min(canvasWidth / width, canvasHeight / height, 1)`，优先更大的可读缩放。
2. 缩放相同时，优先纵横比更接近画布的候选。
3. 再以包围面积和稳定方向作确定性并列判定。

组内选定方向后，沿用现有多分组 shelf packing。整体 packing 的评分同步把最大可读缩放放在纵横比误差之前，避免选出形状接近但实际卡片更小的结果。最终继续调用 `fitRelationshipViewport` 居中并适应窗口，最大自动缩放保持 1，避免小图卡片被过度放大。

### 交互与状态

- “自动排列”继续调用 session 的 `autoArrange(currentCanvas())`，放弃当前节点坐标并保存新布局。
- “适应窗口”只调整 viewport，不移动节点。
- `ResizeObserver` 只更新最近画布尺寸；窗口变化不自动覆盖拖动结果。
- 首次没有缓存时使用当前画布；恢复缓存时保持用户位置，除非用户主动自动排列。

### 边路由

现有曲线边路由支持任意节点坐标，本次不引入新的边数据结构。纵向层级候选可能让上下游表位于不同 x 坐标，仍从相对方向的左右侧连接；自引用、同列和平行外键规则保持不变。布局边界继续包含曲线控制点。

## 错误与性能边界

- 非法或不可见画布仍回退 `960 × 640`。
- 候选出现非有限坐标时沿用自然名称网格回退。
- 每个分组只新增一个线性候选，复杂度保持 O(V + E)；不引入通用布局依赖或随机迭代。
- 五千表链继续使用迭代式强连通分量算法，不增加递归风险。

## 测试与验收

- 新增单一连通/单一名称分组用例，证明宽画布与窄高画布选择不同组内方向，而不仅是多个分组改变列数。
- 断言宽画布布局的宽高比更大、适应后的可读缩放不劣于旧固定方向、所有节点不重叠且结果确定。
- 覆盖不同节点高度、循环、自引用、平行边和五千表链回归。
- session 测试证明 `autoArrange` 使用最新 canvas，`fit` 不移动节点。
- SQLite 与 MySQL Panel 测试继续证明按钮读取当前 canvas，并共享同一 session 行为。
- 运行 relationship-graph 专项测试、SQLite/MySQL 关系图 Panel 测试、类型检查与仓库级检查。

## 验收标准

1. 在典型宽关系图画布中点击“自动排列”，单一大分组不再固定形成狭长竖列，而会显著使用横向空间。
2. 在窄高画布中，相同图会选择更适合纵向空间的稳定布局。
3. 自动排列后所有节点可通过适应窗口看到，节点不重叠，连线有效。
4. SQLite 与 MySQL 行为一致，手工拖动和已缓存布局不会被普通窗口 resize 擅自重置。
