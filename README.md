# MySQL Kit

MySQL Kit 提供面向远程 MySQL 数据库的中文工作台，包括连接管理、对象浏览、数据编辑、结构查看、全库关系图和 SQL 执行。

Kit 由六个职责独立的插件组成：`mysql-core` 独占连接池与全部数据库访问，`mysql-explorer` 管理连接和对象选择，`mysql-data`、`mysql-schema`、`mysql-relationships`、`mysql-sql` 分别提供四个可独立装载的工作区。默认布局将资源管理器固定在左侧，四个工作区使用框架原生标签组展示。

## 连接与安全

- 连接时填写主机、端口、用户和密码；database 可留空，先连接服务器并从左侧选择当前账号可访问的数据库。也可以填写 database 直接进入目标库。
- 密码只保留在当前服务端进程的活动连接配置中；连接成功后 Panel 立即清空密码输入框，连接快照和广播不包含密码，断开连接时清除配置。
- Core 在候选连接探测成功后才替换旧连接；失败重连不会丢失原连接。
- 选择数据库同样使用候选连接池原子切换；选择失败时继续保留原数据库和连接。
- Kit workspace 负责安装 `mysql2` 运行时依赖；六个插件中只有 `mysql-core` 导入驱动并持有连接池。

## 数据与结构

- 数据页支持 25、50、100、250 条分页，并按主键稳定排序。
- 普通表可以新增；只有存在非二进制主键时才能编辑和删除。视图始终只读。
- BIGINT、DECIMAL、时间、JSON 和 BLOB 使用保留类型信息的序列化格式，避免精度损失或二进制误编辑。
- 结构页展示字段、复合主键、索引、外键和 MySQL 返回的定义 SQL。

## 全库关系图

- 关系图只展示当前 database 的普通表和真实声明的外键，不展示 view，也不根据字段名称推断关系。
- 复合外键按 constraint 聚合；自引用、循环和平行外键分别保留。
- 支持搜索、拖拽平移、滚轮与工具栏缩放、适应窗口，以及可见的非 SVG 关系明细。
- 点击或使用键盘激活节点会选择对应表并打开结构 Panel。

## SQL 工作区

- 每次显式执行一条 SQL，行结果最多显示 500 条。
- mutation 结果展示影响行数、插入 ID 和警告状态。
- SQL 错误不会清空草稿；连接变化会清理旧结果但保留当前草稿。
